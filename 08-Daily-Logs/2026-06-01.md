## Decisions Made

### Architectural
- **Lobster workflows must be self-contained** — they bake in the knowledge, learnings, and skills that a sub-agent would need. No thin one-liners that just call Python.
- **Pre-flight checks are mandatory** — every Lobster workflow verifies Chrome CDP, no duplicate crons, credentials present, Gmail folders exist BEFORE running the main script.
- **YAML multiline Python is bad** — use external helper scripts. Lobster's YAML validator chokes on `body='''...'''` triple-quotes and complex regex.

### Data
- **Layer B was the actual blocker** — not embeddings, not author attribution. Without `job_number` + `company_name`, the dataset is text-only and unanalyzable.
- **918 missing jobs are unrecoverable** — they're deleted/archived in SM8. The `job_uuid` is stored but SM8 returns nothing.
- **Staff attribution backfill is deferred** — 94.6% of notes have NULL `staff_uuid` because the SM8 historical backfill didn't include `edit_by_staff_uuid`. This is a SM8 data quality issue, not a code issue.

### Security
- **DB password out of scripts** — moved to `~/.pgpass` file (PostgreSQL convention). The password redaction in the display layer is what forced this, and it's actually a good practice.
- **No hardcoded QB credentials in workflows** — they're referenced as file paths.
