## Executive Summary

The diary database contains **20,266 notes** spanning **17 months** (2025-01-07 to 2026-06-01). However, the dataset has **critical structural gaps** that prevent deep analytical work:

1. **94.7% of notes have no author** (NULL staff_uuid) — only 1,068 of 20,266 have an author
2. **0 notes have job_number populated** — the entire dataset is missing the most important context
3. **0 notes have company_name populated** — cannot group notes by client
4. **0 notes have company_uuid populated** — cannot link to ServiceM8 company records

These gaps mean that today the dataset is **text-only**, with author for the 2026-05-15+ notes and no client/job linkage anywhere. Any analysis is therefore a partial picture.

## 10. Identified Critical Gaps (The Real "Backfilling" Work)

The diary backfill is NOT complete. The README from 2026-05-28 was too optimistic. Here's what still needs doing:

### Gap 1: Author Resolution (HIGHEST PRIORITY)
- 19,198 notes (94.7%) have NULL `staff_uuid` and `staff_member`
- These are all from 2025-01-07 to 2026-05-14 (the historical backfill period)
- The SM8 API must have been returning notes without `edit_by_staff_uuid`
- **Need to:** Either re-fetch with different field selection, or accept that historical notes will remain anonymous
- **Impact:** Cannot do any meaningful staff-pattern analysis on 95% of dataset

### Gap 2: Job Number Resolution (HIGH PRIORITY)
- 0 of 20,266 notes have `job_number` populated
- Watcher stores `related_object_uuid` (UUID) but not the human-readable `generated_job_id`
- **Need to:** Either:
  - Add a job-lookup call to SM8 API per note (slow, expensive)
  - Backfill from existing SM8 job data via separate API call
  - Modify watcher to do a bulk job fetch
- **Impact:** Cannot link diary notes to job cards in SM8. Cannot build per-job analysis.

### Gap 3: Company Resolution (HIGH PRIORITY)
- 0 of 20,266 notes have `company_name` or `company_uuid`
- Same root cause as Gap 2 — notes link to job, but we don't link job to company
- **Need to:** After resolving job_number, also resolve company_uuid → company_name
- **Impact:** Cannot do per-client analysis. Cannot identify high-touch clients.

### Gap 4: Classification Refinement
- 32% of notes are classified as "other" — too high
- 0 notes are classified as "unanswered_tag" — suspicious
- 47% are "direct_response" — probably too high
- **Need to:** Sample 200 "other" classifications, refine rules, re-classify
- **Impact:** Better signal for report triggers

### Gap 5: 29 Empty-Text Notes
- 29 notes have NULL or empty `raw_text`
- These are old (mostly 2025-Q3/Q4) — likely API issues
- **Need to:** Try to re-fetch from SM8, or document as unrecoverable
- **Impact:** Marginal — 0.14% of dataset
